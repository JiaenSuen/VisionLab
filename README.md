## About VisionLab

**VisionLab** is a curated collection of small-scale computer vision experiments, model reproductions, implementation studies, and proof-of-concept systems developed primarily with **PyTorch**.

The repository documents my technical exploration across visual recognition, dense prediction, vision-language learning, geometric vision, and efficient deep learning. Each project is intended to investigate a specific model, technique, dataset, or implementation concept.

> [!NOTE]
> Projects in VisionLab are intentionally compact in scope.
> Larger research projects, publication-related work, and complete systems are maintained as standalone repositories on my GitHub profile.

### Objectives

* Reproduce and study influential computer vision methods
* Test architectural and training ideas through controlled experiments
* Connect theoretical concepts with practical implementations
* Document observations, limitations, and technical takeaways
* Maintain a structured archive of continued computer vision exploration

## Project Labels

Each project is assigned a type according to its primary purpose.

| Label                | Meaning                                                          |
| -------------------- | ---------------------------------------------------------------- |
| **Reproduction**     | Reimplementation or study of an existing method or paper         |
| **Benchmark**        | Controlled comparison between models, methods, or configurations |
| **Study**            | Focused technical investigation of a computer vision concept     |
| **Prototype**        | Early implementation used to test an idea or system design       |
| **Applied System**   | Practical computer vision implementation for a specific use case |
| **Coursework**       | Academic assignment retained for technical documentation         |
| **Proof of Concept** | Minimal experiment used to verify technical feasibility          |

### Status

| Symbol | Status       | Description                                          |
| :----: | ------------ | ---------------------------------------------------- |
|    ✅   | Complete     | Implementation and documentation are available       |
|   🚧   | In Progress  | Active development or additional experiments planned |
|   📦   | Archived     | Retained for reference but not actively maintained   |
|   🧪   | Experimental | Exploratory implementation with unstable interfaces  |


> [!TIP]

## Standard Project Structure

Each documented project should ideally follow a consistent structure:

```text
project-name/
│
├── README.md                   # Objective, method, results, and takeaways
├── requirements.txt            # Python dependencies
├── configs/                    # Training or experiment configurations
├── data/                       # Dataset instructions or sample data
├── models/                     # Model definitions
├── scripts/                    # Training, evaluation, or preprocessing scripts
├── notebooks/                  # Optional analysis notebooks
├── results/                    # Metrics, figures, and prediction examples
└── checkpoints/                # Optional model weights or download instructions
```

A project README should answer five questions:

1. **What is being investigated?**
2. **Why was this experiment conducted?**
3. **How was it implemented?**
4. **What result or observation was obtained?**
5. **What was learned from the experiment?**
  
## Running an Experiment

Because individual projects may use different dependencies or datasets, refer to the README inside each project directory before running it.

A typical workflow is:

```bash
git clone https://github.com/JiaenSuen/VisionLab.git
cd VisionLab

cd path/to/project
pip install -r requirements.txt
python train.py
````

For downloading only a specific project directory, a directory-download service may be used:

[Download a GitHub Directory](https://download-directory.github.io/)

> [!IMPORTANT]
> Verify each project's Python version, dependency list, dataset instructions, and hardware requirements before execution.
